# 双十一模板设计稿

**目标**

为营销页面编辑器增加一个可直接套用的“双十一活动模板”，让用户不必从空白画布开始搭建；模板套用后仍然可继续拖拽、改字、换图、保存和预览。

**设计原则**

- 模板抽象为 `page schema` 预设，而不是写死页面 DOM
- 第一版只做一个双十一单页活动落地页，验证模板链路
- 复用现有 `Text`、`Image`、`Button` 组件，不新增复杂组件
- 模板入口、模板数据、模板应用逻辑分层，便于后续扩展更多营销模板

**为什么不用写死 DOM**

- 写死页面 DOM 更像单独手写一个活动页组件，难以接入现有编辑器数据流
- `page schema` 预设本质上是一份 `PageData`，天然兼容当前渲染、保存、预览和导出链路
- 模板套用后得到的是普通页面数据，用户仍可以继续编辑，而不是只能看一个固定成品页

**MVP 范围**

- 编辑器顶部新增 `使用模板` 入口
- 弹窗中展示一个模板卡片：`双十一活动模板`
- 点击后将模板 schema 注入当前页面
- 如果当前页面已有内容，弹确认提示避免误覆盖
- 套用模板后支持继续编辑、保存草稿、预览、导出 JSON

**模板内容**

- 页面类型：双十一单页活动落地页
- 画布建议尺寸：`1200 x 1600`
- 色彩风格：黑底、红色强调、金色点缀

**页面区块**

1. 顶部活动条
2. 主标题
3. 副标题 / 活动时间
4. 主视觉图
5. 卖点文案区
6. CTA 按钮
7. 活动说明

**交互设计**

- 入口位置：编辑器顶部右侧，和 `新建页面` 同层级
- 用户流程：
  1. 点击 `使用模板`
  2. 选择 `双十一活动模板`
  3. 如当前页面已有内容，先确认是否覆盖
  4. 套用模板并进入可编辑状态

**数据与架构设计**

- 新增模板目录，存放营销模板数据工厂
- 模板以工厂函数形式返回全新的 `PageData`
- Store 新增模板套用方法，负责：
  - 深拷贝模板数据
  - 重新生成页面 id
  - 重新生成组件 id
  - 清空当前选中组件
  - 清空历史记录，避免撤销到旧页面

**边界场景**

- 当前页面已有内容：弹确认框
- 模板图片不可用：先用占位图 URL，后续可替换
- 模板套用后：所有组件仍按普通组件处理，保证可编辑性

**面试表达**

- 模板不是写死的活动页，而是 `page schema` 预设
- 空白创建、模板创建、草稿恢复，本质上都是 `currentPage` 的不同来源
- 通过统一的数据协议复用现有编辑器渲染能力，符合低代码平台思路
